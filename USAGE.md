# Mac GPU Monitor 使用说明

## 简介
这是一个用于监控 Apple Silicon Mac (M1/M2/M3) GPU 使用情况的命令行工具。

## 功能特点
- 自动检测芯片型号（M1/M2/M3系列）
- 显示 GPU 核心数量
- 实时显示 GPU 使用率（进度条）
- 显示显存使用情况（当前值/最大值）
- 记录并显示最近60秒的 GPU 使用历史（柱形图）

## 系统要求
- macOS 11.0 或更高版本
- Apple Silicon Mac（M1/M2/M3 芯片）
- Go 1.22 或更高版本（编译时需要）
- 管理员权限（运行时需要）

## 安装和编译

### 1. 克隆或下载代码
```bash
cd /path/to/your/directory
```

### 2. 编译程序
```bash
go build -o mac_gpu main.go
```

或者使用提供的构建脚本：
```bash
./build.sh
```

### 3. 运行程序
```bash
sudo ./mac_gpu
```

**重要**: 程序需要管理员权限才能获取 GPU 使用信息，因为使用了 `powermetrics` 命令。

## 使用方法

### 启动监控
```bash
sudo ./mac_gpu
```

### 退出程序
按 `Ctrl+C` 退出程序。

## 权限说明

程序需要管理员权限的原因：
- 使用了 `powermetrics` 命令获取 GPU 使用信息
- `powermetrics` 是一个需要 sudo 权限的系统工具
- 该命令可以访问硬件层面的性能数据

如果您不使用 sudo 运行程序：
- 程序会显示警告信息
- 提示您使用 sudo 运行
- GPU 使用率和显存信息将显示为 0
- 芯片型号和核心数仍然可以正常显示

**详细说明**: 为什么要 sudo？请查看 [FAQ.md](FAQ.md) 了解完整的技术解释和安全考虑。

## 故障排除

### 1. 提示 "powermetrics must be invoked as the superuser"
**解决方法**: 使用 sudo 运行程序
```bash
sudo ./mac_gpu
```

### 2. 无法获取 GPU 信息
**可能原因**:
- 没有使用 sudo 权限
- 系统版本过低（需要 macOS 11.0+）

**解决方法**:
- 确保使用 sudo 运行
- 检查系统版本

### 3. 显示的芯片型号不正确
**可能原因**:
- `system_profiler` 命令返回的数据格式变化

**解决方法**:
- 更新程序以支持新的系统版本
- 或者手动编辑代码中的芯片识别逻辑

### 4. 编译失败
**可能原因**:
- Go 版本过低
- 网络问题导致无法下载依赖

**解决方法**:
- 检查 Go 版本：`go version`
- 确保网络连接正常
- 手动下载依赖：`go mod tidy`

## 注意事项

1. **性能影响**: 监控程序本身会占用少量 CPU 资源
2. **历史数据**: 程序只保存最近 60 秒的数据
3. **刷新频率**: 数据每秒更新一次
4. **终端兼容性**: 程序使用 ANSI 转义码清除屏幕，需要在支持 ANSI 转义码的终端中运行
