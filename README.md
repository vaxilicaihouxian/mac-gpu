## 功能特性

### 核心功能
✅ 自动检测 Apple M1/M2/M3 芯片
✅ 显示 GPU 核心数量
✅ 显示 GPU 使用率（基于真实 powermetrics 数据）
✅ 显示 GPU 频率分布（显示 GPU 在不同频率下的工作时间占比）
✅ 显示 GPU 使用历史（最近 60 秒）
✅ 显示显存总量
✅ 现代化、美观的界面
✅ 动态颜色编码

### 界面设计
- **Unicode 边框系统**：圆角和双线边框
- **动态颜色编码**：
  - 🟢 绿色（< 30%）：低负载，安全
  - 🟡 黄色（30-70%）：中等负载，正常
  - 🔴 红色（> 70%）：高负载，注意
- **图标系统**：
  - 🍎 芯片型号
  - 💻 核心数量
  - 💾 显存总量
  - 📊 GPU 使用率
  - 📈 GPU 频率分布
  - 📊 GPU 使用历史

## 技术栈

### 语言和库
- **语言**: Go 1.22
- **依赖**: 标准库（fmt, os, exec, regexp, strings, time）
- **无第三方 UI 库**：保持轻量级和快速启动

### 数据来源
- **system_profiler**: 获取芯片型号和核心数量
- **powermetrics**: 获取 GPU 使用率和频率分布（需要 sudo 权限）

## 设计原则

1. **简单** - 无外部依赖，代码清晰易读
2. **优雅** - 结构良好，逻辑清晰，易于维护
3. **现代** - Unicode 边框，动态颜色，图标系统
4. **真实** - 所有数据来自系统命令，无模拟数据

## 为什么不用分核心显示？

Apple Silicon GPU 采用统一架构，不支持分核心查询。原 Python 版本的"分核心"是随机数模拟，不是真实数据。

**GPU 频率分布** 是更有意义的真实数据：
- 显示 GPU 在不同频率下的工作时间占比
- 帮助理解 GPU 的负载和功耗状态
- 数据直接来自 powermetrics

## 权限要求

程序需要 **sudo 权限** 才能获取 GPU 使用信息：
- `powermetrics` 命令要求管理员权限
- 这是 macOS 的安全设计，保护硬件性能数据不被未经授权的程序访问

## 故障排除

### 常见问题

**问题**: 程序无法获取 GPU 使用率

**解决**:
1. 确保使用 sudo 运行程序
2. 查看 powermetrics 命令是否正常工作
3. 检查 macOS 版本是否兼容

**问题**: 编译错误或运行时错误

**解决**:
1. 确保安装 Go 1.22+: `go version`
2. 清理并重新编译: `go clean && go build -o mac_gpu main.go`
3. 如有问题，运行 `go mod tidy`

**问题**: 界面显示异常

**解决**:
1. 确保终端支持 Unicode: 终端应该使用 UTF-8 编码
2. 检查终端宽度: 程序设计为 80 列宽度
3. 尝试不同的终端模拟器
